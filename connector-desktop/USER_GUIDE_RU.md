# Structura Connector — инструкция для нового пользователя

Пошаговый гайд: где взять данные, что заполнить в окне, какие кнопки нажимать и как проверить подключение.

## 1. Где лежат материалы

Все материалы находятся здесь:

- `https://disk.yandex.ru/d/RZAm7Xh7T4KqwQ`

В этой папке есть:

- установщик `Connector.Desktop.Setup.msi`
- набор `txt` файлов, по одному на каждого пользователя

Важно: нужно открыть именно свой `txt` файл (с вашим `device_id` или ФИО в имени/содержимом).

## 2. Что взять из своего txt файла

Из своего файла скопируйте значения полей:

- `device_id`
- `token`
- `smb_login`
- `smb_password`
- `smb_share_unc`

Важно:

- берите данные только из своего файла (не из файла коллеги)
- `txt` файл на Яндекс.Диске можно открыть прямо в браузере и скопировать значения построчно

Пример:

- `device_id: gpsm-dv`
- `token: ...`
- `smb_login: 62.113.36.107\bim_gpsm_dv`
- `smb_password: ...`
- `smb_share_unc: \\62.113.36.107\BIM_Models`

## 3. Проверка доступа до запуска программы

Откройте PowerShell и выполните:

```powershell
Test-NetConnection server.structura-most.ru -Port 443
curl -UseBasicParsing https://server.structura-most.ru/health
Test-NetConnection server.structura-most.ru -Port 445
```

Ожидаемый результат:

- для `443` и `445`: `TcpTestSucceeded : True`
- для `curl .../health`: `StatusCode : 200` и ответ `{"ok":true}`

Если не получилось, сразу отправьте администратору внешний IP:

```powershell
(Invoke-WebRequest -UseBasicParsing https://api.ipify.org).Content.Trim()
```

## 4. Установка приложения

1. Запустите `Connector.Desktop.Setup.msi`.
2. Завершите установку.
3. Откройте `Structura Connector`.

## 5. Что и куда заполнять в Structura Connector

Для быстрого режима достаточно заполнить:

- `URL сервера`
- `Токен устройства`

Остальные поля могут подставиться автоматически после кнопки `Подключиться (по токену, автоматически)`.

### Блок «Настройки устройства»

- `URL сервера` -> `https://server.structura-most.ru`
- `ID устройства` -> `device_id` из вашего txt
- `Токен устройства` -> `token` из вашего txt
- `Интервал heartbeat (сек)` -> `60`
- `Запускать Structura Connector при входе в Windows` -> рекомендуется включить

### Блок «SMB доступ к BIM_Models»

- `Логин SMB` -> `smb_login` из txt
- `Пароль SMB` -> `smb_password` из txt
- `Путь папки` -> `smb_share_unc` из txt (обычно `\\62.113.36.107\BIM_Models`)

Примечание по логину SMB:

- рекомендуемый формат: полный логин `62.113.36.107\\имя_пользователя`
- если укажете только `имя_пользователя`, приложение обычно подставит сервер автоматически

## 6. Правильный порядок нажатия кнопок

Рекомендуемый быстрый сценарий (одна кнопка):

1. Вставьте `Токен устройства` из своего txt
2. Нажмите `Подключиться (по токену, автоматически)`

Приложение автоматически:

- получит `device_id` и SMB-данные с сервера
- добавит/обновит ваш IP в whitelist через серверную логику
- выполнит вход в SMB-папку
- сохранит настройки
- включит автоотправку heartbeat

Если нужен ручной сценарий:

1. `Проверить подключение к серверу`
2. `Сохранить токен и настройки`
3. `Войти в SMB-папку`
4. `Открыть папку BIMModels`
5. `Включить автоотправку`

Пояснение:

- кнопка `Включить автоотправку` обязательна, чтобы клиент регулярно отправлял heartbeat и сервер подтверждал связь клиента по токену
- после шага 5 окно можно просто закрыть: приложение продолжит работу в трее

## 7. Где хранить модели и как это работает

Рабочее пространство для моделей:

- `\\62.113.36.107\BIM_Models`

Это общее серверное файловое пространство. Все пользователи работают под своими SMB-логинами и паролями.

Поведение как на обычном файловом сервере:

- если файл уже открыт/занят другим пользователем, он блокируется
- Windows показывает, кем занят файл

Модели должны храниться именно в `\\62.113.36.107\BIM_Models`.

## 8. Полный выход из программы

При закрытии окно сворачивается в трей и продолжает работать.

Чтобы полностью выйти:

1. найдите иконку `Structura Connector` в трее
2. нажмите ПКМ
3. выберите `Закрыть`

## 9. Что отправлять администратору при проблеме

Скопируйте и выполните, затем отправьте полный вывод:

```powershell
Test-NetConnection server.structura-most.ru -Port 443
curl -UseBasicParsing https://server.structura-most.ru/health
Test-NetConnection server.structura-most.ru -Port 445
(Invoke-WebRequest -UseBasicParsing https://api.ipify.org).Content.Trim()
```

Также приложите:

- скрин окна `Structura Connector` с заполненными полями (без токена и пароля в открытом виде)
- последние строки из блока `Журнал (последние события)`
